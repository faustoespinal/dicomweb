Query

Query based on ID for DICOM Objects (QIDO) enables you to search for studies, series and instances by patient ID, and receive their unique identifiers to retrieve the binary representations with WADO. QIDO is currently awaiting letter ballot. The specification, as written, can be found here.

Methods

Following are a list of methods accepted by a QIDO service. Click on a method for further detail.

get /studies?...

  • Look up studies (i.e., for a particular patient)

Parameters

Parameter Description Data Type
{dicomAttribute} The field to filter results by. See note
includefield Includes the field to return without filtering on it. {dicomAttribute} | all
fuzzymatching See conformance statement for details. boolean
limit Limits the results to the specified number. integer
offset Skips the first number of specified results. integer

Headers

Header Description Accepted Values
Accept The representation scheme being posted to the RESTful service. multipart/related; type=application/dicom+xml;
application/json
Cache-control If included, specifies that search results returned should be current and not cached. no-cache
